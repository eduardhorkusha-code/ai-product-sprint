## Правила проектування схеми

- Іменування: `snake_case`
- PK: `id UUID PRIMARY KEY DEFAULT gen_random_uuid()` або `BIGINT GENERATED BY DEFAULT AS IDENTITY`
- Зовнішні ключі: завжди явно (наприклад, `user_id UUID REFERENCES auth.users`)
- Індекси: на всі стовпці у WHERE/JOIN: `CREATE INDEX ON habits(user_id);`
- RLS **ОБОВ'ЯЗКОВО** на всіх публічних таблицях
- 1 таблиця = 1 сутність (не денормалізуй)

## SQL — Таблиця habits (habit tracker)

```sql
CREATE TABLE habits (
  id         UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID        NOT NULL REFERENCES auth.users,
  name       TEXT        NOT NULL,
  emoji      TEXT        DEFAULT '✅',
  color      TEXT        DEFAULT '#6366f1',
  is_done    BOOLEAN     NOT NULL DEFAULT FALSE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX ON habits(user_id);
```

## SQL — Таблиця habit_logs (completedDates)

```sql
CREATE TABLE habit_logs (
  id         UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  habit_id   UUID        NOT NULL REFERENCES habits(id) ON DELETE CASCADE,
  user_id    UUID        NOT NULL REFERENCES auth.users,
  completed_date DATE    NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(habit_id, completed_date)
);
CREATE INDEX ON habit_logs(habit_id);
CREATE INDEX ON habit_logs(user_id, completed_date);
```

## SQL — Таблиця moods (Team Pulse)

```sql
CREATE TABLE moods (
  id         UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  device_id  TEXT        NOT NULL,  -- анонімний, без auth
  emoji      TEXT        NOT NULL,
  score      INT         NOT NULL CHECK (score BETWEEN 1 AND 5),
  session_id TEXT        NOT NULL,  -- ідентифікатор сесії голосування
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
-- Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE moods;
```

## SQL — Таблиця win_journal

CREATE TABLE wins (

  text       TEXT        NOT NULL,

## RLS — Вмикання та базові правила

```sql
-- Завжди спочатку вмикаємо RLS
ALTER TABLE public.habits ENABLE ROW LEVEL SECURITY;

-- Базові права
GRANT SELECT ON public.habits TO anon;
GRANT SELECT, INSERT, UPDATE, DELETE ON public.habits TO authenticated;
```

**⚠️ Після вмикання RLS без політик — доступ заборонено всім.**

## 5 RLS-патернів (всі потрібні)

### 1. Owner-only (найчастіший — habit tracker, wins)
```sql
-- Користувач бачить/змінює тільки свої записи
CREATE POLICY "Users view own habits"
  ON habits FOR SELECT TO authenticated
  USING (auth.uid() = user_id);

CREATE POLICY "Users insert own habits"
  ON habits FOR INSERT TO authenticated
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users update own habits"
  ON habits FOR UPDATE TO authenticated
  USING (auth.uid() = user_id);

CREATE POLICY "Users delete own habits"
  ON habits FOR DELETE TO authenticated
  USING (auth.uid() = user_id);
```

### 2. Role-based (адмін бачить все, інші — своє)
```sql
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

CREATE POLICY admins_view_all ON projects FOR SELECT TO authenticated
  USING (EXISTS (
    SELECT 1 FROM user_roles
    WHERE user_id = auth.uid() AND role = 'admin'
  ));

CREATE POLICY users_view_own ON projects FOR SELECT TO authenticated
  USING (
    auth.uid() = owner_id OR
    EXISTS (
      SELECT 1 FROM project_members
      WHERE project_id = projects.id AND user_id = auth.uid()
    )
  );
```

### 3. Public + Auth (blogs, posts)
```sql
-- Всі бачать публічні
CREATE POLICY public_posts ON posts FOR SELECT
  USING (is_public = TRUE);

-- Автор бачить свої приватні
CREATE POLICY private_posts ON posts FOR SELECT TO authenticated
  USING (auth.uid() = author_id);
```

### 4. Multi-tenant (SaaS ізоляція)
```sql
ALTER TABLE tenant_data ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation ON tenant_data FOR ALL TO authenticated
  USING (tenant_id = (
    SELECT tenant_id FROM users WHERE id = auth.uid()
  ));
```

### 5. Anonymous voting (Team Pulse — без auth)
```sql
-- Анонімний INSERT (device_id на клієнті)
ALTER TABLE moods ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Anyone can insert mood"
  ON moods FOR INSERT TO anon, authenticated
  WITH CHECK (true);

CREATE POLICY "Anyone can read moods"
  ON moods FOR SELECT TO anon, authenticated
  USING (true);
```

## Типові помилки RLS

| Помилка | Причина | Фікс |
|---------|---------|------|
| Таблиця повністю відкрита | Забули `ENABLE ROW LEVEL SECURITY` | `ALTER TABLE t ENABLE ROW LEVEL SECURITY;` |
| Всі заблоковані | RLS увімкнено, але політик немає | Додати хоча б одну `CREATE POLICY` |
| JOIN "провалює" фільтр | Суміжна таблиця без RLS | Перевірити всі таблиці у JOIN |
| Повільні запити | USING не використовує індекс | Індексувати стовпці в USING |
| `USING (true)` — занадто широко | Відкриває всі рядки | Замінити на конкретну умову |

## Міграції та CI/CD

```bash
# НЕ роби DDL напряму в продакшн!

# Workflow
supabase db diff > migration.sql          # diff локального стану
supabase db push --file migration.sql     # застосувати
supabase db reset                         # скинути локальну БД

# У Git: зберігай migrations/ і зливай від service_role
supabase db push  # з правами service_role
```

## Flow: додати звичку (sequence)

```
User → [Add Habit screen]
  → Zustand: addHabit() (оновлює локально)
  → FlashList: оновлюється UI миттєво
  → HTTP INSERT → Supabase
  → RLS: auth.uid() = user_id ✓
  → DB: INSERT виконано
  → Повернення результату → (за потреби rollback через undo)
```

## Supabase JS — Швидкий reference

```typescript
// SELECT з фільтром
const { data, error } = await supabase
  .from('habits')
  .select('*')
  .eq('user_id', userId)
  .order('created_at', { ascending: false })

// INSERT
const { data, error } = await supabase
  .from('habits')
  .insert({ name: 'Morning water', emoji: '💧', user_id: userId })
  .select()
  .single()

// UPDATE
const { error } = await supabase
  .from('habits')
  .update({ is_done: true })
  .eq('id', habitId)
  .eq('user_id', userId)  // завжди додавай user_id для безпеки

// DELETE
const { error } = await supabase
  .from('habits')
  .delete()
  .eq('id', habitId)

// Realtime subscription
const channel = supabase
  .channel('moods')
  .on('postgres_changes', {
    event: 'INSERT', schema: 'public', table: 'moods',
    filter: `session_id=eq.${sessionId}`
  }, (payload) => {
    setMoods(prev => [...prev, payload.new as Mood])
  })
  .subscribe()

// Cleanup
return () => supabase.removeChannel(channel)
```
